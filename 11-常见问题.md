# 第十一章：常见问题

## 11.1 安装和运行问题

### 11.1.1 Bun 版本问题

**问题**: 运行时报错 "Bun version not supported"

**解答**: 请确保使用 Bun 1.2.0 或更高版本。运行以下命令升级：

```bash
bun upgrade
```

查看当前版本：

```bash
bun --version
```

### 11.1.2 依赖安装失败

**问题**: bun install 失败或卡住

**解答**: 尝试以下步骤：清除缓存后重试：`rm -rf node_modules bun.lockb && bun install`；检查网络连接；尝试使用国内镜像源。

### 11.1.3 TypeScript 编译错误

**问题**: 看到大量 TypeScript 错误

**解答**: 这是预期的。这是反编译项目，约有 1341 个类型错误来自反编译过程，不影响运行时执行。运行以下命令验证程序可以正常运行：

```bash
bun run dev --version
```

### 11.1.4 Windows 兼容性问题

**问题**: 在 Windows 上运行有问题

**解答**: 确保使用支持 ANSI 颜色的终端，如 Windows Terminal 或 VS Code 终端。避免使用 cmd.exe，建议使用 PowerShell 或 Git Bash。

## 11.2 使用问题

### 11.2.1 API 密钥问题

**问题**: 报错 "API key not found" 或 "Invalid API key"

**解答**: 设置 ANTHROPIC_API_KEY 环境变量：

```bash
# Linux/Mac
export ANTHROPIC_API_KEY=your_api_key

# Windows
set ANTHROPIC_API_KEY=your_api_key
```

或者在项目根目录创建 .env 文件：

```
ANTHROPIC_API_KEY=your_api_key
```

获取 API 密钥：访问 https://console.anthropic.com/ 创建 API 密钥。

### 11.2.2 网络连接问题

**问题**: 无法连接到 Anthropic API

**解答**: 检查以下几点：网络能否访问 api.anthropic.com；是否配置了代理；防火墙是否阻止了连接。

如果需要代理，在环境变量中设置：

```bash
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080
```

### 11.2.3 响应速度慢

**问题**: Claude Code 响应很慢

**解答**: 可以尝试以下优化：使用更快的模型（如 haiku）；减少对话历史长度（使用 /clear 新对话）；检查网络延迟。

### 11.2.4 权限被拒绝

**问题**: 工具调用被拒绝

**解答**: 工具执行需要用户授权。检查：是否在权限提示中选择了拒绝；权限模式是否设置为 dontAsk。

使用更高权限模式：

```bash
bun run dev --dangerously-skip-permissions
```

### 11.2.5 文件操作失败

**问题**: 无法读取或编辑文件

**解答**: 检查：文件路径是否正确；文件权限是否足够；文件是否被其他程序占用。

## 11.3 功能问题

### 11.3.1 工具不可用

**问题**: 某些工具无法使用

**解答**: 工具可能被功能标志禁用。在 cli.tsx 中，feature() 函数始终返回 false，所以很多实验性工具不可用。这是设计决定，不影响核心功能。

### 11.3.2 MCP 连接失败

**问题**: 无法连接到 MCP 服务器

**解答**: 检查以下几点：MCP 服务器是否正在运行；配置是否正确（检查 .mcp.json）；服务器地址是否正确；网络是否连通。

查看 MCP 日志获取详细信息：

```bash
bun run dev --verbose
```

### 11.3.3 会话无法恢复

**问题**: 关闭后无法恢复之前的对话

**解答**: Claude Code 会话信息存储在 ~/.claude/ 目录。确保：没有删除会话文件；使用相同的会话 ID（通过 --resume 指定）。

### 11.3.4 内存占用高

**问题**: 内存占用越来越高

**解答**: 长时间运行后，内存可能会增加。尝试：定期重启程序；使用 /clear 开始新对话；减少 maxTokens 设置。

## 11.4 开发问题

### 11.4.1 代码格式问题

**问题**: lint 或 format 命令报错

**解答**: 使用自动修复：

```bash
bun run lint:fix
bun run format
```

### 11.4.2 构建失败

**问题**: bun run build 失败

**解答**: 检查错误信息。常见原因包括：TypeScript 错误（但实际上应该能构建）；磁盘空间不足；依赖问题。尝试：

```bash
bun install
bun run build
```

### 11.4.3 新工具不生效

**问题**: 创建新工具后无法使用

**解答**: 确保：工具已添加到 src/tools.ts 的 ALL_TOIGHTS 数组；工具名称没有冲突；schema 定义正确。

### 11.4.4 热重载不工作

**问题**: 修改代码后没有生效

**解答**: 开发模式下代码应该自动重载。如果不工作：确认在开发模式运行（bun run dev，不是构建后的版本）；检查是否有语法错误。

## 11.5 配置问题

### 11.5.1 配置文件格式错误

**问题**: 配置文件无效

**解答**: JSON 格式必须是有效的。常见错误：多余逗号；引号未配对；使用单引号（JSON 只支持双引号）。

使用 JSON 验证工具检查：

```bash
cat ~/.claude/settings.json | python -m json.tool
```

### 11.5.2 权限配置不生效

**问题**: 设置的权限规则没有效果

**解答**: 检查：配置文件的路径是否正确；配置格式是否正确；是否有更高优先级的配置覆盖。

### 11.5.3 环境变量不生效

**问题**: 设置的环境变量没有效果

**解答**: 确保：环境变量在运行前设置；在正确的位置设置（系统/用户/当前会话）。

查看当前环境变量：

```bash
env | grep ANTHROPIC
```

## 11.6 性能问题

### 11.6.1 启动慢

**问题**: 程序启动需要很长时间

**解答**: 首次启动需要下载依赖，之后应该很快。可以使用 --version 快速检查。

### 11.6.2 对话卡顿

**问题**: 输入响应不及时

**解答**: 可能原因包括：网络延迟；对话历史过长；终端渲染性能问题。尝试：减少对话历史；使用更快模型。

### 11.6.3 内存泄漏

**问题**: 长时间运行后变慢

**解答**: 这可能是代码问题。尝试：定期重启；报告 Issue 帮助改进。

## 11.7 其他问题

### 11.7.1 项目与官方版本冲突

**问题**: 同时安装了官方版本和此项目

**解答**: 可以同时安装两者，它们使用不同的命令。此项目使用 `bun run dev` 或 `claude-js`（构建后），官方版本使用 `claude`。

**解答**: 可以通过以下方式获取帮助：查看本文档；查看项目 README；搜索现有 Issue；提交新 Issue；查看社区讨论。

### 11.7.3 贡献代码

**问题**: 想为项目贡献代码

**解答**: 欢迎贡献！请阅读开发指南章节，然后：Fork 项目；创建功能分支；提交 Pull Request。
