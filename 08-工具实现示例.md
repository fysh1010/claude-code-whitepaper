# 第八章：工具实现示例

## 8.1 工具实现概述

本章通过分析几个具体工具的实现，帮助您深入理解工具系统的工作原理。我们将详细查看 BashTool、FileEditTool 和 GrepTool 的实现，这些工具分别代表了命令执行、文件编辑和代码搜索三种典型场景。

通过学习这些示例，您将了解：工具的输入验证如何定义；工具的执行逻辑如何实现；工具的权限检查如何工作；工具的结果如何格式化；以及工具的 UI 如何渲染。

## 8.2 BashTool 命令执行工具

### 8.2.1 工具定义

**文件**: `src/tools/BashTool/BashTool.ts`

BashTool 是在终端中执行命令的工具，是 Claude Code 最核心的工具之一。

```typescript
export const BashTool = buildTool({
  name: 'bash',
  description: 'Execute a bash command in the terminal. Use this for running ' +
    'commands, scripts, or other terminal operations.',

  // 输入模式定义
  inputSchema: z.object({
    command: z.string()
      .describe('The bash command to execute'),
    description: z.string()
      .optional()
      .describe('A brief description of what this command does'),
    workdir: z.string()
      .optional()
      .describe('The working directory for the command'),
  }),

  // 结果大小限制
  maxResultSizeChars: 50_000,

  // 执行函数
  async execute(input, context) {
    const { command, workdir } = input;
    const cwd = workdir ?? context.cwd ?? process.cwd();

    try {
      // 执行命令
      const result = await execAsync(command, {
        cwd,
        timeout: 120000,  // 2 分钟超时
        env: { ...process.env, ...context.env },
      });

      return {
        content: [{
          type: 'text',
          text: result.stdout || result.stderr || '(no output)',
        }],
      };
    } catch (error) {
      return {
        content: [{
          type: 'text',
          text: `Error: ${error.message}`,
        }],
        isError: true,
      };
    }
  },

  // 权限检查
  async checkPermissions(input, context) {
    const { command } = input;

    // 检查危险命令
    const dangerousPatterns = [
      /^rm\s+-rf/,
      /^dd\s+/,
      /^mkfs/,
      /^:(){ :|:& };:/,  // Fork bomb
    ];

    for (const pattern of dangerousPatterns) {
      if (pattern.test(command)) {
        return {
          behavior: 'deny',
          message: 'Refusing to execute dangerous command',
        };
      }
    }

    // 默认询问
    return { behavior: 'ask', message: 'Run this bash command?' };
  },
});
```

### 8.2.2 核心执行逻辑

BashTool 的执行逻辑包含以下关键部分：参数解析从输入中提取命令和工作目录；环境设置合并全局环境变量和上下文环境变量；命令执行使用 execAsync 执行命令并设置超时；结果处理将 stdout 和 stderr 转换为标准结果格式；错误处理捕获执行过程中的异常并返回错误信息。

### 8.2.3 权限检查策略

BashTool 的权限检查采用了黑名单模式，预先定义了一些危险命令模式。当用户尝试执��这些命令时，会直接拒绝。常见的危险命令包括递归删除（rm -rf）、磁盘写入（dd）、文件系统格式化（mkfs）以及 fork 炸弹等。

对于其他命令，默认会询问用户权限。这种设计在安全性和便利性之间取得了平衡。

### 8.2.4 结果格式

BashTool 返回的结果格式如下：

```typescript
// 成功结果
{
  content: [{
    type: 'text',
    text: 'command output here',
  }],
}

// 错误结果
{
  content: [{
    type: 'text',
    text: 'Error: command failed',
  }],
  isError: true,
}
```

结果可以包含多个内容块，每个内容块可以是文本、图像或其他类型。

## 8.3 FileEditTool 文件编辑工具

### 8.3.1 工具定义

**文件**: `src/tools/FileEditTool/FileEditTool.ts`

FileEditTool 用于编辑现有文件的内容，是代码修改的核心工具。

```typescript
export const FileEditTool = buildTool({
  name: 'Edit',
  description: 'Make edits to an existing file. Use this to modify code, ' +
    'configuration files, or any text-based file.',

  inputSchema: z.object({
    file_path: z.string()
      .describe('The path to the file to edit'),
    previous_string: z.string()
      .describe('The exact text to replace (must match exactly)'),
    replacement_string: z.string()
      .describe('The text to replace it with'),
    override_goal: z.string()
      .optional()
      .describe('Optional description of the edit goal'),
  }),

  maxResultSizeChars: 10_000,

  async execute(input, context) {
    const { file_path, previous_string, replacement_string } = input;

    try {
      // 读取文件
      let content = await readFile(file_path, 'utf-8');

      // 检查是否匹配
      if (!content.includes(previous_string)) {
        return {
          content: [{
            type: 'text',
            text: `Error: previous_string not found in file`,
          }],
          isError: true,
        };
      }

      // 执行替换
      content = content.replace(previous_string, replacement_string);

      // 写回文件
      await writeFile(file_path, content, 'utf-8');

      return {
        content: [{
          type: 'text',
          text: `Edited ${file_path}`,
        }],
      };
    } catch (error) {
      return {
        content: [{
          type: 'text',
          text: `Error: ${error.message}`,
        }],
        isError: true,
      };
    }
  },

  async checkPermissions(input, context) {
    const { file_path } = input;

    // 检查是否为系统文件
    if (isSystemFile(file_path)) {
      return {
        behavior: 'deny',
        message: 'Cannot edit system files',
      };
    }

    // 检查文件是否存在
    if (!await fileExists(file_path)) {
      return {
        behavior: 'deny',
        message: 'File does not exist',
      };
    }

    return { behavior: 'ask', message: 'Edit this file?' };
  },
});
```

### 8.3.2 精确替换机制

FileEditTool 使用精确字符串匹配来执行编辑，这种设计有几个优点：确定性，给定相同的输入，总是产生相同的结果；安全性，只有精确匹配的文本才会被替换；简单性，不需要理解代码语法就能执行编辑。

不过，这种方式也有局限性：如果文件已经被修改，原来的 previous_string 可能不再匹配。工具会检查匹配并返回错误。

### 8.3.3 路径验证

FileEditTool 在执行权限检查时，会验证：文件是否存在；文件是否为系统文件（不可编辑）；用户是否有权限修改文件。这些检查在执行前进行，避免了对系统文件的意外修改。

### 8.3.4 结果反馈

```typescript
// 成功编辑
{
  content: [{
    type: 'text',
    text: 'Edited src/main.tsx',
  }],
}

// 替换失败
{
  content: [{
    type: 'text',
    text: 'Error: previous_string not found in file',
  }],
  isError: true,
}
```

## 8.4 FileReadTool 文件读取工具

### 8.4.1 工具定义

**文件**: `src/tools/FileReadTool/FileReadTool.ts`

FileReadTool 用于读取文件内容，是获取代码和配置信息的主要工具。

```typescript
export const FileReadTool = buildTool({
  name: 'Read',
  description: 'Read the contents of a file. Use this to view source code, ' +
    'configuration files, or any text-based file.',

  inputSchema: z.object({
    file_path: z.string()
      .describe('The path to the file to read'),
    offset: z.number()
      .optional()
      .describe('Line number to start reading from (1-based)'),
    limit: z.number()
      .optional()
      .describe('Maximum number of lines to read'),
  }),

  maxResultSizeChars: 100_000,

  async execute(input, context) {
    const { file_path, offset, limit } = input;

    try {
      // 读取文件
      let content = await readFile(file_path, 'utf-8');
      const lines = content.split('\n');

      // 应用偏移和限制
      const startLine = (offset ?? 1) - 1;  // 转换为 0 索引
      const endLine = limit ? startLine + limit : lines.length;

      const selectedLines = lines.slice(startLine, endLine);
      const selectedContent = selectedLines.join('\n');

      // 如果有截断，添加说明
      let resultText = selectedContent;
      if (endLine < lines.length) {
        resultText += `\n\n... ${lines.length - endLine} more lines`;
      }
      if (offset && offset > 1) {
        resultText = `... ${startLine} lines skipped ...\n${resultText}`;
      }

      return {
        content: [{
          type: 'text',
          text: resultText,
        }],
      };
    } catch (error) {
      return {
        content: [{
          type: 'text',
          text: `Error: ${error.message}`,
        }],
        isError: true,
      };
    }
  },
});
```

### 8.4.2 分页读取

FileReadTool 支持分页读取，可以指定起始行和最大行数。这种设计对于大型文件非常有用：用户可以先读取文件的开头部分，了解整体结构；如果需要更多内容，可以再读取后续部分。

### 8.4.3 大文件处理

工具设置了 maxResultSizeChars: 100_000 的限制，防止读取过大的文件导致内存问题。当文件超过限制时，会自动截断并添加提示。

## 8.5 GrepTool 代码搜索工具

### 8.5.1 工具定义

**文件**: `src/tools/GrepTool/GrepTool.ts`

GrepTool 用于在代码库中搜索文本，是理解代码结构的重要工具。

```typescript
export const GrepTool = buildTool({
  name: 'grep',
  description: 'Search for text patterns in files. Use this to find function ' +
    'definitions, imports, or any code patterns across your codebase.',

  inputSchema: z.object({
    pattern: z.string()
      .describe('The regex pattern to search for'),
    path: z.string()
      .describe('The file or directory to search in'),
    include: z.string()
      .optional()
      .describe('Glob pattern for files to include (e.g., "*.ts")'),
    exclude: z.string()
      .optional()
      .describe('Glob pattern for files to exclude (e.g., "node_modules")'),
    context: z.number()
      .optional()
      .describe('Number of context lines to show'),
    max_results: z.number()
      .optional()
      .describe('Maximum number of results'),
  }),

  maxResultSizeChars: 50_000,

  async execute(input, context) {
    const { pattern, path, include, exclude, context: contextLines, max_results } = input;

    try {
      // 构建 grep 命令
      const args = ['-n', '-r', '--null'];

      if (contextLines) args.push(`-C ${contextLines}`);
      if (max_results) args.push(`-m ${max_results}`);
      if (include) args.push(`--include=${include}`);
      if (exclude) args.push(`--exclude-dir=${exclude}`);

      args.push(pattern);
      args.push(path || '.');

      // 执行搜索
      const result = await execAsync(`grep ${args.join(' ')}`, {
        cwd: context.cwd,
      });

      // 解析结果
      const matches = parseGrepOutput(result.stdout);

      return {
        content: [{
          type: 'text',
          text: formatGrepResults(matches),
        }],
      };
    } catch (error) {
      if (error.status === 1) {
        // 没有匹配结果
        return {
          content: [{
            type: 'text',
            text: 'No matches found',
          }],
        };
      }

      return {
        content: [{
          type: 'text',
          text: `Error: ${error.message}`,
        }],
        isError: true,
      };
    }
  },
});
```

### 8.5.2 搜索参数

GrepTool 支持丰富的搜索参数：pattern 是正则表达式模式；path 是搜索路径，可以是文件或目录；include 是文件包含模式（如 *.ts）；exclude 是排除模式（如 node_modules）；context 是上下文行数；max_results 是最大结果数。

### 8.5.3 结果格式化

搜索结果的格式化需要考虑可读性，通常会显示：文件名和行号；匹配的代码行；上下文行（如果指定）。格式化时会使用颜色标记匹配的内容，并确保结果在终端中正确显示。

## 8.6 GlobTool 文件查找工具

### 8.6.1 工具定义

**文件**: `src/tools/GlobTool/GlobTool.ts`

GlobTool 用于通过 glob 模式查找文件。

```typescript
export const GlobTool = buildTool({
  name: 'glob',
  description: 'Find files matching a glob pattern. Use this to quickly ' +
    'locate files in your project.',

  inputSchema: z.object({
    pattern: z.string()
      .describe('The glob pattern (e.g., "**/*.ts", "src/**/*.js")'),
    path: z.string()
      .optional()
      .describe('The directory to search in'),
  }),

  maxResultSizeChars: 50_000,

  async execute(input, context) {
    const { pattern, path } = input;
    const searchPath = path || context.cwd || '.';

    try {
      // 使用 glob 库查找文件
      const files = await glob(pattern, {
        cwd: searchPath,
        absolute: false,
        nodir: true,
      });

      if (files.length === 0) {
        return {
          content: [{
            type: 'text',
            text: 'No files found',
          }],
        };
      }

      // 格式化结果
      const resultText = files
        .sort()
        .map(f => f)
        .join('\n');

      return {
        content: [{
          type: 'text',
          text: `${files.length} files found:\n${resultText}`,
        }],
      };
    } catch (error) {
      return {
        content: [{
          type: 'text',
          text: `Error: ${error.message}`,
        }],
        isError: true,
      };
    }
  },
});
```

### 8.6.2 Glob 模式示例

常见的 glob 模式包括：**/*.ts 查找所有 TypeScript 文件；src/**/*.js 查找 src 目录下所有 JS 文件；**/node_modules/** 排除 node_modules 目录；*.test.ts 查找所有测试文件。

export const WebSearchTool = buildTool({
  name: 'websearch',
  description: 'Search the web for information. Use this to find ' +

## 8.8 TaskCreateTool 任务创建工具

### 8.8.1 工具定义

**文件**: `src/tools/TaskCreateTool/TaskCreateTool.ts`

TaskCreateTool 用于在 Claude Code 中创建任务。

```typescript
export const TaskCreateTool = buildTool({
  name: 'TaskCreate',
  description: 'Create a new task. Use this to track work that needs to be done.',

  inputSchema: z.object({
    title: z.string()
      .describe('The task title'),
    description: z.string()
      .optional()
      .describe('Detailed task description'),
    priority: z.enum(['low', 'medium', 'high'])
      .optional()
      .describe('Task priority'),
  }),

  maxResultSizeChars: 1_000,

  async execute(input, context) {
    const { title, description, priority = 'medium' } = input;

    // 创建任务
    const task = {
      id: generateTaskId(),
      title,
      description,
      priority,
      status: 'pending',
      createdAt: Date.now(),
    };

    // 保存到状态
    const setAppState = useSetAppState();
    setAppState(prev => ({
      ...prev,
      tasks: {
        ...prev.tasks,
        [task.id]: task,
      },
    }));

    return {
      content: [{
        type: 'text',
        text: `Created task: ${title} (${task.id})`,
      }],
    };
  },
});
```
