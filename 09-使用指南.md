# 第九章：使用指南

## 9.1 快速开始

### 9.1.1 环境要求

Claude Code 需要以下环境才能运行：Bun 运行时，版本要求 1.2.0 或更高，建议使用最新版本；终端环境，支持 ANSI 颜色的终端应用，如 iTerm2、Windows Terminal 等；网络访问，能够连接到 Anthropic API（api.anthropic.com）。

建议在运行前执行 bun upgrade 确保使用最新版本的 Bun。

### 9.1.2 安装步骤

首先克隆项目仓库：

```bash
git clone https://github.com/claude-code-best/claude-code.git
cd claude-code
```

然后安装项目依赖：

```bash
bun install
```

构建项目（可选，用于生产环境）：

```bash
bun run build
```

### 9.1.3 运行 Claude Code

开发模式运行（推荐用于开发测试）：

```bash
bun run dev
```

使用构建后的版本：

```bash
./dist/cli.js
```

或者添加到 PATH 后直接使用：

```bash
claude-js --version
```

## 9.2 基础交互

### 9.2.1 启动对话

运行 Claude Code 后，会看到 REPL 界面，底部有输入提示符。在提示符后输入您的问题或请求，然后按 Enter 发送。

```bash
$ bun run dev
Claude Code 2.1.888

你可以直接输入问题或请求，我会尽力帮助你。

> 帮我解释一下什么是闭包
```

### 9.2.2 退出对话

有几种方式可以退出对话：输入 exit 并按 Enter；按 Ctrl+C 两次；输入 quit 并按 Enter。

### 9.2.3 查看帮助

输入 help 查看可用命令和快捷键：

```
> help
```

## 9.3 命令行参数

### 9.3.1 常用参数

Claude Code 支持多种命令行参数，用于控制启动行为。

查看版本：

```bash
bun run dev --version
# 或
bun run dev -v
```

打印模式（处理输入后输出结果，适合脚本）：

```bash
echo "解释这段代码" | bun run dev -p
# 或
bun run dev --print < file.txt
```

指定工作目录：

```bash
bun run dev /path/to/project
```

指定模型：

```bash
bun run dev --model claude-sonnet-4-20250514
```

### 9.3.2 权限模式

默认模式，每次工具调用都会请求权限：

```bash
bun run dev
```

自动接受编辑（不询问文件编辑权限）：

```bash
bun run dev --dangerously-accept-edits
```

跳过所有权限检查（仅限可信环境）：

```bash
bun run dev --dangerously-skip-permissions
```

计划模式，只在计划阶段请求权限：

```bash
bun run dev --plan
```

### 9.3.3 工作树模式

Claude Code 支持 Git worktree，可以为每个任务创建独立的 Git 目录：

```bash
# 基于 PR 创建 worktree
bun run dev --worktree --pr 123

# 基于分支创建 worktree
bun run dev --worktree --branch feature-branch

# 指定 worktree 名称
bun run dev --worktree --name my-task
```

## 9.4 REPL 交互技巧

### 9.4.1 输入模式

默认的输入模式是普通文本模式。您可以直接输入问题或请求。

以斜杠（/）开头的输入会被视为命令：

```
> /help
> /clear
> /resume
```

### 9.4.2 快捷键

Claude Code 支持多种键盘快捷键，提高操作效率：

Ctrl+O：切换到 Transcript 模式（查看完整对话历史）
Ctrl+C：取消当前操作
Ctrl+L：清屏
Ctrl+U：清除当前输入行

在 Transcript 模式下：/：开始搜索；n：跳到下一个匹配；N：跳到上一个匹配；Esc：退出搜索

### 9.4.3 自动补全

在输入过程中，Claude Code 会尝试自动补全：文件名补全；命令补全；代码补全（使用 Tab）

### 9.4.4 消息导航

Transcript 模式允许您浏览完整的对话历史。使用箭头键上下滚动，使用 / 搜索特定内容。

## 9.5 常用工作流

### 9.5.1 代码审查

让 Claude Code 审查代码是常见的使用场景：

```
> 审查 src/main.tsx 中的代码
```

审查会包括：代码质量问题；潜在的安全风险；性能优化建议；代码风格问题。

### 9.5.2 缺陷修复

描述您遇到的错误，让 Claude Code 帮助诊断和修复：

```
> 这个函数抛出了 "undefined is not a function" 错误，请帮我修复
```

Claude Code 会分析错误信息，定位问题代码，并提供修复方案。

### 9.5.3 代码生成

让 Claude Code 生成代码：

```
> 帮我写一个 React 组件，用于显示用户列表
```

可以指定更多细节：

```
> 帮我写一个 TypeScript 函数，接受一个字符串数组，返回按字母排序的字符串
```

### 9.5.4 解释代码

让 Claude Code 解释难以理解的代码：

```
> 解释一下这个函数做了什么
# 然后粘贴代码
```

### 9.5.5 重构代码

请求 Claude Code 重写代码：

```
> 重构这段代码，使用 async/await
```

### 9.5.6 运行测试

让 Claude Code 运行和调试测试：

```
> 运行测试
# 或
> 运行 src/__tests__/example.test.ts
```

### 9.5.7 Git 操作

Claude Code 可以帮助执行 Git 操作：

```
> 创建并切换到新分支
> 提交当前的修改
> 查看最近的提交历史
```

## 9.6 配置

### 9.6.1 全局配置

全局配置文件位于 ~/.claude/settings.json（Linux/Mac）或 %USERPROFILE%\.claude\settings.json（Windows）。

示例配置：

```json
{
  "permissions": {
    "allow": ["BashTool:git*", "FileReadTool"],
    "deny": ["BashTool:rm -rf"]
  },
  "model": "claude-sonnet-4-20250514",
  "maxTokens": 4096
}
```

### 9.6.2 项目配置

项目级配置放在 .claude/settings.json 中：

```json
{
  "permissions": {
    "allow": ["FileEditTool:src/**"]
  },
  "hooks": {
    "PreCommit": ["bun run lint"]
  }
}
```

### 9.6.3 MCP 配置

MCP 服务器配置放在 .mcp.json 中：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/directory"]
    }
  }
}
```

## 9.7 权限系统使用

### 9.7.1 权限提示

当工具需要执行可能敏感的操作时，会显示权限请求。权限请求会显示：操作描述（如“编辑文件 src/app.ts”）；操作详情（如“替换 10 行代码”）；允许、拒绝、编辑选项。

### 9.7.2 权限选项

在权限提示中，您可以：Allow：允许执行本次操作；Allow & Remember：允许并记住此决定（未来相同操作自动允许）；Deny：拒绝执行；Edit：修改参数后重试。

### 9.7.3 权限模式选择

根据使用场景选择合适的权限模式：学习环境使用默认模式，每次确认确保安全；CI/CD 环境使用 --dangerously-skip-permissions 提高效率；计划模式使用 --plan，先查看计划再执行。

## 9.8 管道和脚本

### 9.8.1 管道输入

使用管道将内容发送给 Claude Code：

```bash
# 从文件读取
cat problem.txt | bun run dev -p

# 从标准输入
echo "解释这段代码" | bun run dev -p
```

### 9.8.2 输出重定向

将输出重定向到文件：

```bash
bun run dev -p < question.txt > answer.txt
```

### 9.8.3 脚本中使用

在脚本中使用 Claude Code：

```bash
#!/bin/bash
RESULT=$(echo "$1" | bun run dev -p)
echo "$RESULT"
```

## 9.9 故障排除

### 9.9.1 常见错误

**API 密钥错误**：确保 ANTHROPIC_API_KEY 环境变量已设置。

**网络连接问题**：检查是否能访问 api.anthropic.com。

**权限被拒绝**：检查文件权限设置。

### 9.9.2 调试模式

使用 --verbose 查看详细日志：

```bash
bun run dev --verbose
```

### 9.9.3 查看日志

日志文件位置：Linux/Mac: ~/.claude/logs/；Windows: %USERPROFILE%\.claude\logs/。

### 9.9.4 报告问题

如果遇到 bug，请提交 Issue：https://github.com/claude-code-best/claude-code/issues

请包含：问题描述；复现步骤；相关日志；环境信息。

## 9.10 最佳实践

### 9.10.1 提示技巧

清晰描述需求：不要只说“修复这个”，而要说“修复这个导致数组越界的 bug”。

提供上下文：说明代码的用途、您尝试过的方法等。

分步进行：复杂任务可以分成多个小步骤。

### 9.10.2 安全建议

谨慎使用 --dangerously-skip-permissions：这会绕过所有安全检查。

审查工具调用：即使使用自动模式，也应该审查工具调用请求。

保护敏感信息：不要在对话中暴露 API 密钥、密码等敏感信息。

### 9.10.3 性能优化

使用计划模式处理复杂任务：先查看计划，再决定是否执行。

利用 MCP 扩展功能：对于频繁使用的工具，可以配置 MCP 服务器。

保持对话简洁：过长的对话历史会影响响应速度和准确性。
