# 第五章：工具系统

## 5.1 工具系统概述

工具系统是 Claude Code 与外部世界交互的核心机制。它使得 Claude Code 能够执行文件操作、运行命令、搜索代码、访问网络等。工具系统采用了统一的接口设计，使得添加新工具变得标准化和简单。目前 Claude Code 内置了超过五十个工具，涵盖了编程工作中的各种常见任务。

工具系统的设计遵循几个核心原则。首先是接口统一性，所有工具都遵循相同的接口规范，包括名称、描述、输入模式、执行函数和权限检查。其次是权限可控性，每个工具都可以定义自己的权限检查逻辑，确保在用户授权的范围内执行。第三是 UI 独立性，工具可以选择自定义渲染逻辑，在界面中显示特定格式的输出。第四是可扩展性，新增工具只需要遵循接口规范，不需要修改核心代码。

## 5.2 工具接口定义

### 5.2.1 Tool 类型

**文件**: `src/Tool.ts`

Tool.ts 定义了工具的完整接口规范。

```typescript
// 工具基础接口
interface Tool<Input extends z.ZodType> {
  // 工具标识
  name: string;
  description: string;

  // 输入验证
  inputSchema: Input;
  validateInput?: (input: unknown, context: ToolUseContext) => Promise<ValidationResult>;

  // 执行逻辑
  execute(input: z.infer<Input>, context: ToolUseContext): Promise<ToolResult>;

  // 权限检查
  checkPermissions(input: z.infer<Input>, context: ToolUseContext): Promise<PermissionResult>;
  preparePermissionMatcher?(input: z.infer<Input>): Promise<(pattern: string) => boolean>;

  // UI 渲染（可选）
  renderToolUseMessage?(props: ToolUseMessageProps): React.ReactNode;
  renderToolUseProgressMessage?(props: ToolUseProgressProps): React.ReactNode;
  renderToolResultMessage?(props: ToolResultMessageProps): React.ReactNode;

  // 工具特性
  isMcp?: boolean;  // 是否为 MCP 工具
  isOpenWorld?: () => boolean;  // 是否为开放世界工具
  maxResultSizeChars?: number;  // 结果大小限制
}
```

每个工具都需要实现这些接口中的核心方法。 execute 是工具的核心执行逻辑， checkPermissions 负责权限验证， inputSchema 定义了输入的 JSON Schema。

### 5.2.2 buildTool 辅助函数

**文件**: `src/Tool.ts` (第 600+ 行)

buildTool 是一个工厂函数，用于简化工具的创建过程。

```typescript
function buildTool<Input extends z.ZodType>(
  config: ToolConfig<Input>
): Tool<Input> {
  return {
    name: config.name,
    description: config.description,
    inputSchema: config.inputSchema,
    maxResultSizeChars: config.maxResultSizeChars ?? DEFAULT_MAX_RESULT_SIZE,

    // 执行函数包装
    execute: async (input, context) => {
      // 验证输入
      const validation = config.inputSchema.safeParse(input);
      if (!validation.success) {
        return { error: validation.error.message };
      }

      // 权限检查
      const permission = await config.checkPermissions?.(input, context);
      if (permission?.behavior === 'deny') {
        return { error: permission.message };
      }

      // 执行核心逻辑
      return await config.execute(input, context);
    },

    // 权限检查（默认实现）
    checkPermissions: config.checkPermissions ?? (async () => ({ behavior: 'ask' })),

    // UI 渲染（使用配置或默认）
    renderToolUseMessage: config.renderToolUseMessage ?? defaultRenderToolUseMessage,
    renderToolUseProgressMessage: config.renderToolUseProgressMessage,
    renderToolResultMessage: config.renderToolResultMessage ?? defaultRenderToolResultMessage,

    // 其他特性
    isMcp: config.isMcp,
    isOpenWorld: config.isOpenWorld,
  };
}
```

buildTool 提供了默认的权限检查和错误处理逻辑，使得工具开发者可以专注于核心业务逻辑。

### 5.2.3 工具名称匹配

**文件**: `src/Tool.ts` (工具函数部分)

工具系统提供了几个实用函数用于工具名称匹配。

```typescript
// 查找工具
export function findToolByName(tools: Tool[], name: string): Tool | undefined {
  return tools.find(tool => toolMatchesName(tool, name));
}

// 工具名称匹配（支持别名）
export function toolMatchesName(tool: Tool, name: string): boolean {
  // 直接匹配
  if (tool.name === name) return true;

  // 别名匹配（如果有）
  const aliases = (tool as any).aliases;
  if (aliases && Array.isArray(aliases)) {
    return aliases.includes(name);
  }

  return false;
}
```

```typescript
export const ALL_TOOLS = [
  // 核心工具
  BashTool,
  PowerShellTool,
  FileEditTool,
  FileReadTool,
  FileWriteTool,

  // 搜索工具
  GrepTool,
  GlobTool,

  // 网络工具
  WebSearchTool,
  WebFetchTool,

  // 任务工具
  TaskCreateTool,
  TaskListTool,
  TaskUpdateTool,

  // ... 更多工具
];

// 获取可用工具（考虑功能标志和用户类型）
export function getTools(): Tool[] {
  const userType = process.env.USER_TYPE;

  return ALL_TOOLS.filter(tool => {
    // 根据功能标志过滤
    const gate = getToolGate(tool.name);
    if (gate && !feature(gate)) {
      return false;
    }

    // 根据用户类型过滤
    if (tool.userType && tool.userType !== userType) {
      return false;
    }

    return true;
  });
}
```

工具注册表的设计使得新增工具只需要在 ALL_TOOLS 数组中添加导入和引用，其他逻辑会自动处理。

### 5.3.2 工具分类

工具可以按照功能进行分类：

**命令执行类工具**：BashTool、PowerShellTool 用于在终端中执行命令。

**文件操作类工具**：FileReadTool、FileWriteTool、FileEditTool 用于文件的读取、写入和编辑。

**搜索类工具**：GrepTool、GlobTool 用于在代码库中搜索内容。

**网络类工具**：WebSearchTool、WebFetchTool 用于搜索和获取网络内容。

**任务管理类工具**：TaskCreateTool、TaskListTool、TaskUpdateTool 用于创建和管理任务。

**MCP 类工具**：MCPTool 用于与 MCP 服务器通信。

## 5.4 工具执行模型

### 5.4.1 执行流程概述

工具的执行流程可以分为以下几个阶段：检测阶段检测到 API 响应中包含工具调用；验证阶段验证工具输入是否符合 schema；权限检查阶段检查用户是否授权此操作；执行阶段调用工具的 execute 函数；结果处理阶段将结果格式化并添加到消息历史。

这个流程中，检测和结果处理在 query.ts 中完成，验证和权限检查由工具系统处理，实际执行由各个工具负责。

### 5.4.2 StreamingToolExecutor

**文件**: `src/services/tools/StreamingToolExecutor.ts`

StreamingToolExecutor 是流式工具执行的实现，支持并行执行多个工具。

```typescript
class StreamingToolExecutor {
  private tools: Tool[];
  private canUseTool: CanUseToolFn;
  private context: ToolUseContext;
  private pendingTools: Map<string, ToolUseBlock> = new Map();
  private completedResults: ToolResult[] = [];

  addTool(toolBlock: ToolUseBlock, assistantMessage: AssistantMessage): void {
    // 将工具加入待执行队列
    const tool = findToolByName(this.tools, toolBlock.name);
    if (!tool) {
      this.completedResults.push({
        toolUseId: toolBlock.id,
        error: `Tool not found: ${toolBlock.name}`,
      });
      return;
    }

    // 并行执行
    this.executeTool(tool, toolBlock, assistantMessage);
  }

  private async executeTool(...): Promise<void> {
    // 权限检查
    const permission = await tool.checkPermissions(input, this.context);
    if (permission.behavior === 'deny') {
      // 拒绝执行
      return;
    }

    // 执行工具
    const result = await tool.execute(input, this.context);
    this.completedResults.push(result);
  }
}
```

流式执行的核心优势是多个工具可以同时运行，大大减少了总等待时间。

### 5.4.3 传统工具执行

对于不支持流式执行的场景，工具系统也支持传统的顺序执行。

```typescript
async function runTools(
  toolBlocks: ToolUseBlock[],
  assistantMessages: AssistantMessage[],
  canUseTool: CanUseToolFn,
  toolUseContext: ToolUseContext,
): Promise<ToolResult[]> {
  const results: ToolResult[] = [];

  for (const toolBlock of toolBlocks) {
    const tool = findToolByName(availableTools, toolBlock.name);
    if (!tool) {
      results.push({ toolUseId: toolBlock.id, error: `Tool not found` });
      continue;
    }

    // 验证输入
    const validation = tool.inputSchema.safeParse(toolBlock.input);
    if (!validation.success) {
      results.push({ toolUseId: toolBlock.id, error: validation.error.message });
      continue;
    }

    // 权限检查
    const permission = await tool.checkPermissions(toolBlock.input, toolUseContext);
    if (permission.behavior === 'deny') {
      results.push({ toolUseId: toolBlock.id, error: permission.message });
      continue;
    }

    // 执行
    const result = await tool.execute(toolBlock.input, toolUseContext);
    results.push({ ...result, toolUseId: toolBlock.id });
  }

  return results;
}
```

## 5.5 权限系统

### 5.5.1 权限模式

### 5.5.2 权限规则

**文件**: `src/types/permissions.ts` (第 75-79 行)

权限规则定义了用户对特定工具的偏好设置。

```typescript
type PermissionRule = {
  source: PermissionRuleSource;  // 规则来源
  ruleBehavior: PermissionBehavior;  // 'allow' | 'deny' | 'ask'
  ruleValue: PermissionRuleValue;  // { toolName, ruleContent? }
};

type PermissionRuleSource =
  | 'userSettings'   // 用户设置
  | 'projectSettings' // 项目设置
  | 'localSettings'  // 本地设置
  | 'flagSettings'   // 标志设置

```

权限规则可以从多个来源定义，包括用户配置、项目配置和命令行参数。

### 5.5.3 权限决策流程

**文件**: `src/utils/permissions/permissions.ts` (第 400-500 行)

权限决策是一个多层匹配的过程。

```typescript
async function checkToolPermissions(
  tool: Tool,
  input: unknown,
  context: ToolUseContext,
): Promise<PermissionResult> {
  // 1. 验证输入
  const validation = await tool.validateInput?.(input, context);
  if (validation?.result === false) {
    return { behavior: 'deny', message: validation.message };
  }

  // 2. 工具特定的权限检查
  const toolPermission = await tool.checkPermissions(input, context);

  // 3. 应用权限规则匹配
  const ruleResult = matchPermissionRules(tool, input, context.toolPermissionContext);

  // 4. 根据模式和规则做出最终决策
  return resolvePermissionDecision(toolPermission, ruleResult, context);
}
```

权限决策考虑工具的默认行为、用户的规则设置和当前的权限模式。

### 5.5.4 权限决策类型

**文件**: `src/types/permissions.ts` (第 171-266 行)

```typescript
type PermissionDecision =
  | PermissionAllowDecision    // 允许执行
  | PermissionAskDecision      // 请求用户确认
  | PermissionDenyDecision;   // 拒绝执行

type PermissionResult =
  | PermissionDecision
  | { behavior: 'passthrough', message: string };  // 传递给下一层处理
```

每种决策类型都有对应的处理逻辑：允许执行会直接运行工具；请求确认会显示权限提示 UI；拒绝执行会返回错误。

## 5.6 工具结果处理

### 5.6.1 结果格式

工具执行完成后，结果会被规范化为统一格式。

```typescript
type ToolResult = {
  toolUseId: string;  // 工具调用 ID
  content?: Content;  // 结果内容
  error?: string;     // 错误信息
  isError?: boolean;  // 是否为错误
};
```

内容可以是文本、图像或其他格式，取决于工具的实现。

### 5.6.2 结果大小限制

每个工具都可以定义最大结果大小，防止单个工具输出耗尽内存。

```typescript
// 工具定义中的大小限制
maxResultSizeChars: 100_000,

// 结果截断逻辑
function truncateResult(result: string, maxSize: number): string {
  if (result.length <= maxSize) return result;
  return result.slice(0, maxSize) + `\n\n[Output truncated, ${result.length - maxSize} chars omitted]`;
}
```

当结果超过限制时，会被截断并添加提示信息。

### 5.6.3 结果渲染

工具可以选择自定义渲染方式，在 UI 中显示格式化的输出。

```typescript
// 工具定义中的渲染函数
renderToolUseMessage: (props) => <CustomToolUseMessage {...props} />,
renderToolResultMessage: (props) => <CustomToolResultMessage {...props} />,

// 默认渲染
function defaultRenderToolUseMessage(props) {
  return (
    <Box>
      <Text bold>Running {props.toolName}</Text>
      <Text>{props.inputSummary}</Text>
    </Box>
  );
}
```

自定义渲染允许工具根据输出类型显示不同的 UI，如代码高亮、表格等。

```typescript
export const BashTool = buildTool({
```
