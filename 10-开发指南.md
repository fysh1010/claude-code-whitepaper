# 第十章：开发指南

## 10.1 开发环境设置

### 10.1.1 环境要求

开始开发前，需要确保环境满足以下要求：Bun 运行时，版本 1.2.0 或更高；Git，用于版本控制；代码编辑器，推荐 VS Code 或 WebStorm，并安装 TypeScript 插件。

### 10.1.2 克隆和安装

首先克隆项目仓库：

```bash
git clone https://github.com/claude-code-best/claude-code.git
cd claude-code
```

然后安装依赖：

```bash
bun install
```

### 10.1.3 项目结构概览

项目的主要目录结构如下：

```
claude-code/
├── src/                    # 源代码
│   ├── entrypoints/       # 入口点
│   ├── tools/             # 工具实现
│   ├── components/        # UI 组件
│   ├── services/          # 服务层
│   └── state/             # 状态管理
├── dist/                  # 构建产物
├── docs/                  # 文档
└── packages/              # 工作区包
```

### 10.1.4 开发命令

常用的开发命令：

```bash
# 开发模式（热重载）
bun run dev

# 构建生产版本
bun run build

# 运行 linter
bun run lint

# 自动修复 lint 问题
bun run lint:fix

# 代码格式化
bun run format
```

## 10.2 代码组织

### 10.2.1 模块划分

代码按照功能模块划分，每个模块都有明确的职责：入口模块（src/entrypoints/）负责程序启动和初始化；工具模块（src/tools/）负责实现各种工具功能；界面模块（src/components/ 和 src/screens/）负责用户界面；服务模块（src/services/）负责提供基础设施功能；状态模块（src/state/）负责应用状态管理。

### 10.2.2 命名规范

文件命名使用 kebab-case：my-tool.ts；my-component.tsx。

目录命名也使用 kebab-case：file-edit-tool/；prompt-input/。

TypeScript 类型和接口使用 PascalCase：interface ToolConfig {}；type MessageState = {}。

常量使用 SCREAMING_SNAKE_CASE：const MAX_RECONNECT_ATTEMPTS = 5;

### 10.2.3 导入顺序

代码中的导入应该按照以下顺序：外部库导入（React, zod 等）；内部模块导入（从 src/ 目录）；本地导入（同一目录内的模块）。

```typescript
// 1. 外部库
import { useState, useCallback } from 'react';
import { z } from 'zod';

// 2. 内部模块
import { useAppState } from '../../state/AppState.js';
import { Tool } from '../Tool.js';

// 3. 本地
import { MY_CONSTANT } from './constants.js';
```

## 10.3 调试技巧

### 10.3.1 开发模式

使用开发模式运行程序，可以获得更好的调试体验：

```bash
bun run dev
```

开发模式的特点包括：直接运行源码，无需构建；保留 console.log 输出；更详细的错误堆栈。

### 10.3.2 日志调试

在代码中添加日志是基本的调试方法：

```typescript
// 添加调试日志
console.log('Debug info:', someValue);

// 条件日志
if (process.env.DEBUG) {
  console.log('Verbose:', detailedInfo);
}

// 错误日志
console.error('Error occurred:', error);
```

### 10.3.3 TypeScript 类型检查

运行类型检查以发现类型错误：

```bash
bunx tsc --noEmit
```

### 10.3.4 VS Code 调试配置

创建 .vscode/launch.json 配置调试：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug",
      "type": "node",
      "request": "launch",
      "program": "${workspaceFolder}/src/entrypoints/cli.tsx",
      "runtimeExecutable": "bun",
      "runtimeArgs": ["run", "src/entrypoints/cli.tsx"],
      "console": "integratedTerminal"
    }
  ]
}
```

## 10.4 创建新工具

### 10.4.1 工具目录结构

每个工具都有自己的目录，遵循统一的结构：

```
src/tools/MyTool/
├── index.ts           # 导出工具实例
├── MyTool.ts         # 工具核心实现
├── schema.ts         # 输入 schema（可选）
├── permission.ts     # 权限检查（可选）
└── UI.tsx            # 自定义 UI（可选）
```

### 10.4.2 工具实现步骤

下面以创建一个简单的工具为例，说明实现步骤。

**第一步：创建目录和文件**

创建 src/tools/HelloTool/index.ts 文件。

**第二步：定义工具**

```typescript
import { buildTool } from '../Tool.js';
import { z } from 'zod';

// 使用 buildTool 创建工具
export const HelloTool = buildTool({
  name: 'hello',
  description: 'A simple tool that greets the user',

  // 定义输入 schema
  inputSchema: z.object({
    name: z.string()
      .describe('The name to greet'),
    greeting: z.string()
      .optional()
      .describe('Custom greeting message'),
  }),

  // 设置结果大小限制
  maxResultSizeChars: 1_000,

  // 实现执行逻辑
  async execute(input, context) {
    const { name, greeting = 'Hello' } = input;

    return {
      content: [{
        type: 'text',
        text: `${greeting}, ${name}!`,
      }],
    };
  },

  // 实现权限检查
  async checkPermissions(input, context) {
    // 简单工具不需要特殊权限
    return { behavior: 'ask', message: 'Say hello?' };
  },
});
```

**第三步：注册工具**

在 src/tools.ts 中添加导入：

```typescript
import { HelloTool } from './tools/HelloTool/index.js';
```

在 ALL_TOOLS 数组中添加：

```typescript
export const ALL_TOOLS = [
  // ... 其他工具
  HelloTool,
];
```

### 10.4.3 工具测试

为工具编写测试以确保正确性：

```typescript
import { describe, it, expect } from 'bun:test';
import { HelloTool } from './HelloTool/index.js';

describe('HelloTool', () => {
  it('should greet with default greeting', async () => {
    const result = await HelloTool.execute(
      { name: 'World' },
      { cwd: '/tmp' }
    );

    expect(result.content[0].text).toBe('Hello, World!');
  });

  it('should use custom greeting', async () => {
    const result = await HelloTool.execute(
      { name: 'Alice', greeting: 'Hi' },
      { cwd: '/tmp' }
    );

    expect(result.content[0].text).toBe('Hi, Alice!');
  });
});
```

运行测试：

```bash
bun test
```

## 10.5 扩展 UI 组件

### 10.5.1 添加新组件

在 src/components/ 目录下创建新组件：

```typescript
// src/components/MyComponent/MyComponent.tsx
import React from 'react';
import { Box, Text } from 'ink';

export function MyComponent({ title }: { title: string }) {
  return (
    <Box>
      <Text bold>{title}</Text>
    </Box>
  );
}
```

### 10.5.2 使用组件

在需要的地方导入并使用：

```typescript
import { MyComponent } from '../components/MyComponent/MyComponent.js';

function SomeScreen() {
  return (
    <Box>
      <MyComponent title="Hello" />
    </Box>
  );
}
```

### 10.5.3 Ink 组件注意事项

Ink 组件有一些特殊限制需要注意：只能使用 Ink 提供的组件（如 Box、Text），不能使用 HTML 元素；所有组件必须是函数组件；状态更新使用 useState，不能使用 useEffect 进行副作用。

## 10.6 添加新服务

### 10.6.1 服务目录结构

在 src/services/ 下创建新服务：

```
src/services/my-service/
├── index.ts       # 导出
├── my-service.ts  # 服务实现
└── types.ts       # 类型定义
```

### 10.6.2 服务实现示例

```typescript
// src/services/my-service/index.ts
export class MyService {
  private config: MyServiceConfig;

  constructor(config: MyServiceConfig) {
    this.config = config;
  }

  async initialize(): Promise<void> {
    // 初始化逻辑
  }

  async doSomething(): Promise<Result> {
    // 服务逻辑
    return { success: true };
  }
}

// 工厂函数
export function createMyService(config: MyServiceConfig): MyService {
  return new MyService(config);
}
```

### 10.6.3 在应用中使用服务

在需要的地方导入和使用：

```typescript
import { createMyService } from '../services/my-service/index.js';

const service = createMyService({ /* 配置 */ });
await service.initialize();
const result = await service.doSomething();
```

## 10.7 修改状态管理

### 10.7.1 添加新状态

在 src/state/AppStateStore.ts 中添加新状态：

```typescript
// 添加新状态字段
type AppState = {
  // ... 现有字段
  myFeature: {
    enabled: boolean;
    data: MyData[];
  };
};

// 添加默认值
const DEFAULT_APP_STATE: AppState = {
  // ... 现有默认值
  myFeature: {
    enabled: false,
    data: [],
  },
};
```

### 10.7.2 使用状态

在组件中使用新状态：

```typescript
import { useAppState } from '../state/AppState.js';

function MyComponent() {
  const myFeature = useAppState(s => s.myFeature);

  return (
    <Box>
      <Text>Enabled: {myFeature.enabled.toString()}</Text>
    </Box>
  );
}
```

### 10.7.3 更新状态

更新状态时使用函数式更新：

```typescript
const setAppState = useSetAppState();

setAppState(prev => ({
  ...prev,
  myFeature: {
    ...prev.myFeature,
    enabled: true,
  },
}));
```

## 10.8 构建和发布

### 10.8.1 构建命令

构建生产版本：

```bash
bun run build
```

构建产物位于 dist/ 目录。

### 10.8.2 本地测试构建

测试构建后的版本：

```bash
./dist/cli.js --version
```

### 10.8.3 发布到 npm（可选）

如果想发布自己的版本：

```bash
# 更新 version
npm version patch

# 构建
bun run build

# 发布
npm publish
```

### 10.9.1 提交规范

提交代码时，请遵循 conventional commits 规范：

```bash
# 功能新增
git commit -m "feat: add new tool MyTool"

# bug 修复
git commit -m "fix: resolve issue with tool permissions"

# 文档更新
git commit -m "docs: update usage guide"
```
